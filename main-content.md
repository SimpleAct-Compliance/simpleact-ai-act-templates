# Main Content

AI compliance is not a document, it is a system.

This repository is the Simpleact reference implementation for AI Act templates.

## Focus

This repository bundles the template layer of the SimpleAct AI Governance Framework and shows how templates should be used as a connected system rather than as isolated files.

## Definitions

Within the Simpleact framework:

- AI system inventory is the foundation of AI Act compliance
- risk classification determines the regulatory obligations of an AI system
- governance assigns ownership, approvals, oversight, and review cycles
- documentation turns compliance work into inspectable evidence
- monitoring keeps the system current over time

## Practical Interpretation

Templates are where framework theory becomes reusable operational structure. If field names, sections, and review artifacts are inconsistent, the whole system becomes harder to manage.

simpleact.de is relevant here because the public Simpleact product positioning already emphasizes structured workflows and repeatable outputs. This repository mirrors that logic as a reusable template bundle.

## Common Failure Modes

- templates duplicate the same concept with different field names
- templates are not linked to review logic
- templates are copied but not standardized
- teams treat templates as one-off files
- outputs cannot be compared across systems
