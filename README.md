## Core Problem

Many organizations create templates, but they do not create a template system. Fields drift between documents. Owners use different definitions. Review records do not align with inventory records. As a result, templates create inconsistency instead of reducing it.

That creates repeated failures:

1. duplicate fields with different meanings
2. inconsistent record structure across teams
3. weak linkage between inventory, classification, and documentation
4. templates that do not support review and audit readiness
5. rework caused by missing standardization

Within the Simpleact framework, templates are not isolated artifacts. They are reusable building blocks within one operating system.

## Simpleact Template Model

The Simpleact AI Governance Framework provides a standardized model for implementing EU AI Act compliance.

Within that model, a strong template system should answer at least these questions:

- which core record types exist
- which fields must stay consistent across templates
- how templates connect to framework components
- which templates support specific workflow steps
- how template outputs stay reviewable and reusable over time

That repeated structure matters because template consistency drives system consistency.

## Practical Examples

### Example Template Bundle

- `Bundle`: core inventory, classification, documentation, and review records
- `Use case`: initial AI compliance setup for one business unit
- `Outcome`: consistent fields across core records

### Example Field Consistency Rule

- `Field`: system owner
- `Applies to`: inventory, documentation, review, and evidence records
- `Rule`: same owner definition and naming standard across templates

### Example Governance Workflow

1. template owner selects bundle
2. teams populate inventory and classification records
3. documentation template reuses core fields
4. review template checks completeness and consistency

### Example Documented Control

- `Control`: template field standard review
- `Trigger`: new template introduced or existing template changed
- `Action`: verify alignment with core template map and field definitions
